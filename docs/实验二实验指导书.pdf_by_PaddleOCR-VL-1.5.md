### 4.1 简单工厂模式应用

#### 4.1.1 道具创建场景分析

在飞机大战游戏中包含 5 种道具：加血道具、火力道具、超级火力道具、炸弹道具、冰冻道具。所有道具均在特定敌机被击毁时按预设概率随机掉落生成，生成后以固定速度向屏幕下方移动，英雄机碰撞后自动触发对应效果，不同道具的效果存在差异（如提升火力、恢复生命、清除敌机等）。

#### 简单工厂模式

简单工厂模式（Simple Factory Pattern）属于创建型设计模式，由一个工厂类根据传入的参数，决定创建并返回哪一种产品类实例。它将对象的创建与使用分离，让客户端无需关心对象的创建细节，只需调用工厂方法即可获取所需实例。

#### 4.1.2 代码实现

简单工厂模式的代码示例：

步骤 1：创建一个接口，充当产品角色。

Product.java
public interface Product {
    void show();
}

步骤 2：创建实现接口的实体类，充当具体产品角色。

ConcreteProductA.java
public class ConcreteProductA implements Product {
    @Override
    public void show() {
        System.out.println("I am product A.");
    }
}

}
}

ConcreteProductB.java
public class ConcreteProductB implements Product {
@Override
public void show() {
    System.out.println("I am product B.");
}

步骤 3：创建一个简单工厂类，负责创建产品。

注：简单工厂模式中，如果工厂类没有状态、不需要被继承、只是用来统一创建对象，那么工厂方法可以写成 static。

public class SimpleFactory {
    public static Product createProduct(String type) {
        switch (type) {
            case "A":
                return new ConcreteProductA();
                case "B":
                    return new ConcreteProductB();
                default:
                    throw new IllegalArgumentException("Unknown product type ! ");
            }
        }
    }
}

步骤 4：使用 SimpleFactoryPatternDemo 来演示简单工厂模式的用法

SimpleFactoryPatternDemo.java

public class SimpleFactoryPatternDemo {
    public static void main(String[] args) {
        Product productA = SimpleFactory.createProduct("A");
        productA.show();

        Product productB = SimpleFactory.createProduct("B");
        productB.show();
    }
}

步骤 5：执行程序，输出结果：

I am product A.
I am product B.

### 4.2 工厂方法模式应用

#### 4.2.1 敌机创建场景分析

在飞机大战游戏中，敌机是核心作战单位，目前主要涵盖5种不同类型：普通敌机、精英敌机、精锐敌机、王牌敌机和Boss敌机。不同类型敌机在生命值、攻击力、移动方式、子弹弹道、得分奖励、生成概率等方面均存在明显差异，且游戏运行过程中会持续、随机生成各类敌机，生成逻辑分散且复杂。另外，随着游戏版本的迭代，未来极可能引入更多新型敌机，系统必须具备良好的可扩展性。

设计任务：鉴于敌机种类多样、结构差异明显、创建逻辑复杂、未来可能持续扩展的特性，采用工厂方法模式进行敌机创建类设计与代码重构，将对象创建与使用分离，满足高扩展需求。

UML 建模：绘制该模式的 UML 类图，图中需明确标注设计模式名称，并自定义合理的类名、属性名及方法名，完整展示工厂方法模式的实现结构。

#### 工厂方法模式

工厂方法模式（Factory Method Pattern）也是一种创建型设计模式，其在父类中提供一个创建对象的方法，允许子类决定实例化对象的类型。

工厂方法模式代码示例：

步骤 1：创建一个接口，充当产品角色。

Shape.java
public interface Shape {
    void draw();
}

步骤 2：创建实现接口的实体类，充当具体产品角色。

public class Rectangle implements Shape {

    @Override
    public void draw() {
        System.out.println("Inside Rectangle::draw() method.");
    }
}
Square.java
public class Square implements Shape {

@Override
public void draw() {
    System.out.println("Inside Square::draw() method.");
}

Circle.java

public class Circle implements Shape {

@Override
public void draw() {
    System.out.println("Inside Circle::draw() method.");
}

步骤 3：创建一个工厂接口，充当创建者角色。

ShapeFactory.java
public interface ShapeFactory {
    public abstract Shape createShape();
}

步骤 4：创建实现工厂接口的具体工厂类，充当具体创建者角色。

RectangleFactory.java
public class RectangleFactory implements ShapeFactory {

@Override
public Shape createShape() {
    return new Rectangle();
}

public class SquareFactory implements ShapeFactory {

    @Override
    public Shape createShape() {
        return new Square();
    }
}

CircleFactory.java

public class CircleFactory implements ShapeFactory {

@Override
public Shape createShape() {
    return new Circle();
}

步骤 5：使用 FactoryPatternDemo 来演示工厂方法模式的用法。

FactoryPatternDemo.java
public class FactoryPatternDemo {

    public static void main(String[] args) {
        ShapeFactory shapeFactory;
        Shape shape;

        // 获取 Circle 的对象，并调用它的 draw 方法
        shapeFactory = new CircleFactory();
        shape = shapeFactory.createShape();
        shape.draw();

        // 获取 Rectangle 的对象，并调用它的 draw 方法
        shapeFactory = new RectangleFactory();
        shape = shapeFactory.createShape();
        shape.draw();

        // 获取 Square 的对象，并调用它的 draw 方法
        shapeFactory = new SquareFactory();
        shape = shapeFactory.createShape();
        shape.draw();

    }
}

Inside Rectangle::draw() method.

## 5. 本次迭代开发目标
